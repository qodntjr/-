데이터베이스 - 여러 사람 또는 응용 시스템들이 공유할 목적으로 구조화하여 관리하는 데이터들의 집합
ㄴ구조화 -> 데이타모델-관계형 데이타모델 (oracle mysql msql)
RDBMS - 테이블로 데이터를 저장, FK,PK로 관계를 맺어줌

테이터베이스 관리 시스템의 장점
1. 데이터의 중복성과 불일치성 감소
2. 데이터의 독립성
3. 질의 처리에 효율적인 저장 구조
4. 동시접근
5. 데이터보안
6. 백업 복구
7. 일관된 데이터를 유지
8. 다양한 인터페이스 -> SQL 

데이터베이스 시스템의 데이터 모델
계층형 모델 - 1:n 형태로 제한, 
네트워크형 모델 - 복잡하고 변경이 어렵다, 
관계형 모델, 객체지향형 모델, 객체관계형 모델(ORDBMS)

데이터베이스 관리 시스템의 구조 
스키마 - 틀, 인스턴스 - 데이터
데이터베이스 사용자
관리자(DBA), 설계자, 사용자
데이터베이스 언어
SQL(정의어DDL 조작어DML 제어어DCL)

비정형 데이터베이스
NoSQL 데이터 모델컬럼, 도큐먼트 키 값 그래프

관계형 데이터베이스의 용어와 식별자 유형
1. 관계(테이블 또는 릴레이션) 2. 튜플(레코드 또는 행) 3.애트리뷰트(속성 또는 열)
4. 도메인 5. 슈퍼키, 후보키, 기본키(PK), 대체키, 외래키(FK-다른테이블을 참조)

관계형 데이터베이스에서의 제약조건
1.참조 무결성 제약조건(연결고리가 존재해야함) 2. 키 제약조건(중복X) 
3. 도메인 제약조건(도메인에 속하는 원소)

제약조건의 위배에 따른 처리
1. 제한 2. 연쇄(참조되는 릴레이션, 참조하는 릴레이션 함께 삭제) 3. 널 값 4. 디폴트 값

RDBMS에서 가장 중요하게 여기는 성질은 ACID고, NoSQL에서는 CAP 이론이다.
ACID(Atomicity, Consistency, Isolation, Durability)
